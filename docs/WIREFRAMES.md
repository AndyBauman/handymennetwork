# Layout outlines (key screens)

## Customer — Home

- Sticky header: logo, nav links (no hamburger on desktop), **Bigger text**, **Get help** (amber) — no public phone on site.
- Hero: H1, subhead, two full-width buttons (Browse, Call).
- Trust bar: four checkmarks.
- How it works: three equal cards.
- Popular repairs: 2-column cards with price + Add to list.
- Comparison table.
- Tracking demo module.
- Reviews: three cards.
- Service area chips.
- FAQ accordion (15+).
- Closing CTA band.

## Customer — Browse menu

- H1 + intro.
- Repeat per category: H2, grid of repair cards (name, price, time, included, materials, CTA).

## Customer — Book / checkout (phase 2)

- Cart summary (line items, bundle savings).
- Address + notes + “who will be home”.
- Payment (Stripe).
- Confirmation + “we’re finding a pro” state linking to Track.

## Customer — Track

- Large status headline matching dispatch table.
- Map placeholder or provider widget.
- Assigned visit card (in app only): minimal ID for safe meetup — **no** public pro directory on marketing pages.
- Secondary: Support chat / call.

## Handyman — Dashboard

- Prominent **Active** toggle.
- Incoming offers list (distance, payout, items) — Accept / Pass.
- Current job panel: address, Navigate, status buttons.
- Tabs: Earnings, Reviews, Profile.

## Global

- Footer: Get help + trust/vetting link + book.
- Skip link, focus rings, 48px+ targets.
