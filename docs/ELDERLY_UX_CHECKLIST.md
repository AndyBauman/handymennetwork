# Elderly / non-technical UX checklist

- [ ] Body text **≥ 18px**; headlines clearly larger.
- [ ] Contrast meets **WCAG AA minimum** (AAA where feasible); no light gray on white for body.
- [ ] **Get help** path is obvious (header/footer) — in-app Help when logged in; no reliance on a public phone number on the site.
- [ ] **Bigger text** toggle persists (localStorage).
- [ ] Primary button is the **largest** obvious action; label says what happens (“Add to my list”).
- [ ] No jargon in customer copy (“handyman,” not “provider”; “on the way,” not “dispatched”).
- [ ] Forms: visible **labels**, not placeholder-only; errors in plain language.
- [ ] **Skip to content** link works.
- [ ] Full **keyboard** path through nav, FAQs, and checkout.
- [ ] **Focus** states visible on all interactive elements.
- [ ] `details/summary` FAQs usable with screen readers (test VoiceOver / NVDA).
- [ ] Touch targets **≥ 48px** (prefer 56px).
- [ ] No dead ends — family-assisted booking and email help documented for users who avoid apps.
- [ ] Trust row on booking: **total repeated** (“Your total is $95. That’s it.”).
- [ ] Test with a **real 70+ user** on a mid-range Android phone.
